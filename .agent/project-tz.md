## 1. Введение и общее описание проекта

### 1.1. Цель проекта

Создание SaaS-платформы (Software as a Service) для строительных компаний (застройщиков) Таджикистана и стран СНГ. Платформа предназначена для комплексной автоматизации всех бизнес-процессов застройщика: от приобретения земельного участка и управления строительством до продажи квартир, ведения финансов, управления персоналом и взаимодействия с клиентами.

### 1.2. Бизнес-модель

Платформа работает по модели SaaS с мультитенантной архитектурой. Владелец платформы (Super Admin) регистрирует строительные компании, настраивает тарифные планы и управляет подписками. Каждая строительная компания получает изолированное пространство с полным набором инструментов для управления бизнесом.

### 1.3. Целевая аудитория

- Строительные компании (застройщики) Таджикистана
- Строительные компании стран СНГ (масштабирование)
- Покупатели квартир (мобильное приложение, личный кабинет)
- Риелторские агентства и брокеры-партнёры

### 1.4. Ключевые преимущества (USP)

- Полная локализация под рынок Таджикистана: мультивалютность (сомони, доллары, рубли), SMS-рассылки через местных операторов, учёт специфики покупки земли
- Единая экосистема: CRM + финансы + склад + мастера + клиентский портал + мобильное приложение
- 3D-визуализация фасада здания и интерактивная шахматка квартир
- Автоматическая генерация договоров и графиков платежей
- Мультитенантная SaaS-архитектура с управлением подписками

### 1.5. Аналоги и конкуренты (анализ рынка)

При разработке были изучены следующие решения: Profitbase, Домопланер, JoyWork, SberCRM, 4DEV, G-PLUS CRM, iPROFi house, ALLIO, а также презентация AmarCRM. Ключевые функции конкурентов, которые необходимо реализовать и превзойти:

- Интерактивная шахматка с реал-тайм обновлением статусов квартир
- 3D-фасад здания с кликабельными этажами и квартирами
- Динамическое ценообразование на основе спроса
- Воронка продаж с автоматизацией этапов
- Ипотечный калькулятор и работа с рассрочками
- Генерация документов по шаблонам
- Интеграция с мессенджерами, телефонией, SMS
- Сквозная аналитика и дашборды

### 2.2. Мультитенантная архитектура

Каждая строительная компания получает собственный `tenant_id`. Все данные привязаны к этому идентификатору. На уровне базы данных реализуется Row Level Security (RLS) для гарантии изоляции данных между компаниями.

- Каждая таблица содержит поле `tenant_id`
- RLS-политики автоматически фильтруют данные по текущему тенанту
- API-запросы автоматически привязывают данные к тенанту пользователя
- Резервное копирование и восстановление возможно как для всей системы, так и для отдельного тенанта

### 2.3. Безопасность

- JWT-токены для аутентификации с refresh-token ротацией
- RBAC (Role-Based Access Control) — гибкая система ролей и прав доступа
- Шифрование данных в покое (AES-256) и при передаче (TLS 1.3)
- Двухфакторная аутентификация (2FA) для администраторов
- Аудит-лог всех действий пользователей
- Защита от CSRF, XSS, SQL-инъекций
- Регулярные бэкапы с geo-репликацией

## 3. Роли и права доступа

### 3.1. Иерархия ролей

| Роль | Уровень | Описание |
|------|---------|----------|
| **Super Admin** | Платформа | Владелец SaaS-платформы. Управление всеми застройщиками, тарифами, подписками, глобальными настройками |
| **Админ компании** | Тенант | Руководитель строительной компании. Полный доступ ко всем модулям своей компании |
| **Руководитель отдела продаж (РОП)** | Тенант | Управление менеджерами, сделками, аналитика продаж, KPI |
| **Менеджер по продажам** | Тенант | Работа с клиентами, сделками, бронирование, оформление документов |
| **Бухгалтер** | Тенант | Финансовый учёт, платежи, отчётность, журнал финансов |
| **Кассир** | Тенант | Приём и выдача денег, кассовые операции, касса |
| **Прораб / Завсклад** | Тенант | Учёт стройматериалов, приёмка, списание, инвентаризация |
| **Завсклад** | Тенант | Управление складом, остатками, перемещениями материалов |
| **Брокер/Агент (внешний)** | Тенант | Партнёрский доступ: просмотр шахматки, бронирование, ограниченный функционал |
| **Клиент / Покупатель** | Приложение | Просмотр своих данных, графика платежей, документов, чат с застройщиком |

### 3.2. Матрица прав доступа

Для каждого модуля системы настраивается доступ по принципу CRUD (создание, чтение, обновление, удаление). Админ компании может создавать кастомные роли и гибко настраивать права доступа для каждого сотрудника.

### 3.3. Super Admin — панель управления платформой

Super Admin имеет отдельную панель администрирования, не связанную с тенантами:

- Просмотр списка всех зарегистрированных строительных компаний
- Создание нового тенанта (застройщика) и назначение администратора
- Управление тарифными планами и биллингом
- Блокировка/разблокировка тенантов
- Просмотр общей статистики платформы: количество тенантов, сделок, объём транзакций
- Управление глобальными настройками: SMS-провайдеры, валюты, шаблоны
- Аудит-лог всех критических действий
- Управление обновлениями и миграциями

## 4. Модули системы — детальное описание

### 4.1. Модуль «Управление объектами» (Жилые комплексы)

#### 4.1.1. Создание объекта (ЖК)

Застройщик создаёт жилой комплекс (объект) с полной информацией:

- Название ЖК, адрес, город, район
- Тип объекта: жилой комплекс, коммерческая недвижимость, многоэтажный дом, малоэтажный дом, коттеджный посёлок
- Статус объекта: проектирование, строительство, сдан, приостановлен
- Дата начала и планируемая дата завершения строительства
- Основная валюта расчётов для данного объекта
- Описание объекта, инфраструктура, преимущества
- Фотографии, рендеры, видео-туры
- 3D-модель фасада (файл загружается или генерируется)
- Документы: разрешения, лицензии, проектная документация

#### 4.1.2. Структура объекта: Блоки, Этажи, Помещения

Каждый объект имеет иерархическую структуру:

- **ЖК (Объект)** → содержит один или несколько блоков/секций
- **Блок/Секция** → содержит этажи (включая подземные)
- **Этаж** → содержит помещения (квартиры, офисы, коммерция, паркинг, кладовые)

Для каждого помещения (квартиры) указывается:

- ID помещения (уникальный номер)
- Номер квартиры
- Тип: квартира, студия, пентхаус, офис, коммерческое помещение, паркинг, кладовая
- Количество комнат
- Общая площадь (м²), жилая площадь, площадь кухни, площадь балкона/лоджии
- Этаж и блок
- Цена (базовая, текущая, за м²)
- Статус: свободна, забронирована, продана, зарезервирована (для владельцев земли)
- Особенности планировки: евроремонт, без отделки, чистовая, черновая
- Планировка (изображение, PDF)
- Фотографии (до и после ремонта)
- Вид из окна (описание, фото)
- Расположение по шахматке: слева или справа на этаже

#### 4.1.3. Визуализация: Шахматка квартир

Шахматка — визуальная матрица всех помещений объекта. Это основной инструмент менеджера по продажам для быстрого обзора наличия.

- Режимы отображения: шахматка (сетка), список (таблица), 3D-фасад
- Цветовая кодировка статусов: 🟢 зелёный (свободна), 🟡 жёлтый (бронь), 🔴 красный (продана), ⚪ серый (зарезервирована)
- Фильтры: по типу помещения, количеству комнат, цене, площади, этажу, блоку
- При клике на ячейку — всплывающее окно с полной информацией о квартире
- Возможность добавить квартиру в сделку прямо из шахматки
- Реал-тайм обновление: если другой менеджер забронировал квартиру, она мгновенно меняет цвет для всех пользователей (WebSocket)
- Печать шахматки для офлайн-показа клиентам

#### 4.1.4. 3D-фасад здания

Интерактивный 3D-фасад жилого комплекса для визуальной демонстрации клиентам:

- Загрузка 3D-модели здания (формат glTF/glb) или генерация из параметров (этажи, блоки)
- При наведении на этаж — подсветка и информация: номер этажа, количество свободных квартир по комнатности
- При клике на этаж — переход к шахматке этого этажа
- Цветовая индикация: заселённость этажа (процент проданных квартир)
- Возможность вращения, приближения и отдаления 3D-модели
- Адаптация для мобильных устройств

### 4.2. Модуль «Управление земельным участком»

> ⚠️ Данный модуль учитывает **специфику рынка Таджикистана**, где покупка земли — сложный процесс с множеством договорённостей с разными владельцами.

#### 4.2.1. Реестр земельных участков

- Адрес и кадастровый номер участка
- Площадь участка (м²)
- Целевое назначение: под какой объект (ЖК) приобретается
- Статус: в переговорах, договор заключён, оплата завершена, в собственности

#### 4.2.2. Сделки с владельцами земли/домов

Для каждого владельца ведётся отдельная карточка:

- ФИО владельца, контактные данные, паспортные данные
- Адрес объекта (дом/участок), который приобретается
- **Тип сделки:**
  - 💰 **Денежная компенсация** — согласованная сумма, валюта, график выплат, статус оплат
  - 🏠 **Бартер (квартира из будущего ЖК)** — номер и характеристики обещанной квартиры, статус передачи
  - 🔄 **Комбинированная** — денежная часть + бартерная часть
- Документы: договор купли-продажи, акты, расписки
- История переговоров и комментарии

#### 4.2.3. Финансовый учёт по земле

Все расходы на приобретение земли автоматически отражаются в финансовом модуле как категория расходов «Приобретение земельного участка». Бартерные квартиры учитываются как обязательства застройщика и резервируются в шахматке со статусом «Зарезервирована для владельца земли».

### 4.3. Модуль «Клиенты и CRM»

#### 4.3.1. Карточка клиента

Каждый клиент имеет подробную карточку:

- ФИО, дата рождения, пол
- Паспортные данные (серия, номер, кем и когда выдан)
- ИНН (если применимо)
- Телефоны (основной, дополнительный), WhatsApp, Telegram
- Email
- Адрес проживания
- Источник: откуда узнал (реклама, агент, знакомые, сайт, соц. сети)
- Статус клиента в воронке (см. ниже)
- Привязанный менеджер
- История всех взаимодействий: звонки, встречи, показы, сообщения
- Привязанные сделки, договоры, платежи
- Теги и заметки

#### 4.3.2. Воронка продаж (Sales Pipeline)

Настраиваемая канбан-доска с этапами:

| Этап | Описание |
|------|----------|
| Новый лид | Первый контакт, ещё не обработан |
| В обработке | Менеджер связался, выявляет потребности |
| Показ/Встреча | Назначена или проведена встреча/показ объекта |
| Заинтересован | Клиент выбрал квартиру, обсуждение условий |
| Бронирование | Квартира забронирована за клиентом (с таймером: бронь действует N дней) |
| Оформление сделки | Подготовка и подписание договора |
| Оплата | Полная или первый взнос по рассрочке |
| Завершена | Сделка закрыта, ключи переданы |
| Отказ | Клиент отказался (указать причину) |
| Отложен | Клиент попросил вернуться позже |

Этапы воронки настраиваются администратором компании. Для каждого этапа можно задать автоматические действия: отправка SMS, постановка задачи менеджеру, уведомление руководителю.

#### 4.3.3. Задачи и напоминания

- Автоматические задачи при переходе сделки на новый этап
- Ручное создание задач менеджером
- Привязка задач к клиенту, сделке, объекту
- Уведомления о просроченных задачах
- Канбан-доска задач для каждого менеджера

#### 4.3.4. История взаимодействий

Полная хронология всех контактов с клиентом: входящие/исходящие звонки (с записью, при интеграции с телефонией), SMS-сообщения, WhatsApp-переписка, встречи, показы, комментарии менеджера. Все действия логируются автоматически с временной меткой и авторством.

### 4.4. Модуль «Касса и сделки»

#### 4.4.1. Процесс оформления сделки

Пошаговый процесс продажи квартиры:

1. Менеджер выбирает объект (ЖК) и квартиру из шахматки или списка
2. Выбирает существующего клиента или создаёт нового
3. Выбирает тип оплаты: полная оплата, рассрочка, ипотека
4. Для рассрочки: указывает срок (например, 24 месяца), первоначальный взнос, периодичность платежей
5. Система автоматически рассчитывает график платежей
6. Генерируется договор по шаблону с автоматической подстановкой данных
7. Менеджер распечатывает договор, подписывает с клиентом
8. Фиксируется первый платёж (или полная оплата)
9. Квартира меняет статус на «Продана»
10. Клиенту отправляется SMS-подтверждение

#### 4.4.2. Типы оплаты

| Тип оплаты | Описание | Особенности |
|------------|----------|-------------|
| **Полная оплата** | 100% стоимости квартиры единовременно | Возможна скидка. Один платёж, один документ |
| **Рассрочка от застройщика** | Частичная оплата с графиком | Первоначальный взнос + ежемесячные платежи. Настраиваемый срок (3–60 мес.), возможна наценка за рассрочку |
| **Ипотека** | Через банк-партнёр | Застройщик получает 100% от банка. Ипотечный менеджер фиксирует данные банка, ставку, сумму |
| **Бартер** | Обмен (например, земля на квартиру) | Нет денежных поступлений. Квартира переходит как обязательство |
| **Комбинированная** | Часть деньгами + часть рассрочкой | Гибкая схема, настраивается индивидуально |

#### 4.4.3. Автоматический расчёт графика платежей

При оформлении рассрочки система автоматически генерирует график:

- Общая сумма сделки
- Первоначальный взнос (сумма и процент)
- Остаток к оплате
- Количество платежей
- Периодичность: ежемесячно, ежеквартально, произвольно
- Дата каждого платежа
- Сумма каждого платежа
- Наценка за рассрочку (если применимо, фиксированная или процентная)
- Итоговая сумма с наценкой

> График можно корректировать вручную после генерации (например, неравные платежи).

#### 4.4.4. Автогенерация договоров

Система генерирует договоры из настраиваемых шаблонов:

- Шаблоны создаются в настройках: договор купли-продажи, договор рассрочки, акт приёма-передачи и др.
- В шаблоне используются переменные-плейсхолдеры: `{{client_name}}`, `{{apartment_number}}`, `{{total_price}}`, `{{payment_schedule}}` и т.д.
- При генерации все переменные заменяются реальными данными из сделки
- Генерируется PDF-файл для печати
- Договор сохраняется в карточке сделки и клиента
- Поддержка версионности: если условия изменились, создаётся новая версия

### 4.5. Модуль «Платежи и график оплат»

#### 4.5.1. Приём платежей

Фиксация каждого платежа от клиента:

- ID платежа (автоматический)
- Клиент (из карточки сделки)
- Сделка и квартира
- Сумма платежа
- Валюта платежа (может отличаться от основной)
- Курс конвертации на момент операции
- Сумма в основной валюте (автоматический пересчёт)
- Расчётный счёт / касса, на которую поступил платёж
- Способ оплаты: наличные, банковский перевод, мобильный платёж
- Дата платежа
- Статус: принят, на проверке, подтверждён, отклонён
- Квитанция / чек (генерируется автоматически)
- Ответственный кассир/бухгалтер

#### 4.5.2. График платежей клиента

Для каждой сделки с рассрочкой ведётся интерактивный график:

- Таблица: дата платежа, плановая сумма, фактически оплачено, остаток, статус
- Цветовая индикация: 🟢 зелёный (оплачен), 🟡 жёлтый (приближается), 🔴 красный (просрочен), ⚪ серый (будущий)
- Автоматические SMS-напоминания за N дней до платежа (настраивается)
- SMS-уведомление при успешной оплате: сумма, остаток долга, дата следующего платежа
- Автоматическое начисление пени за просрочку (настраивается: процент, льготный период)
- Возможность частичной оплаты (закрывает часть суммы платежа)
- Возможность досрочного погашения

#### 4.5.3. Реестр всех платежей

Общий журнал всех поступлений по всем сделкам с фильтрами: по объекту, по клиенту, по дате, по менеджеру, по статусу. Экспорт в Excel/PDF. Статистика: сумма поступлений за период, задолженность, прогноз поступлений.

### 4.6. Модуль «Финансы и бухгалтерия»

#### 4.6.1. Журнал финансов (главная книга)

Центральный модуль учёта всех денежных операций компании:

- Каждая операция: дата, тип (доход/расход/перевод), категория, сумма, валюта, счёт, описание, ответственный
- **Доходы:** оплата от клиентов, возвраты, инвестиции, прочие поступления
- **Расходы:** покупка земли, стройматериалы, оплата мастеров, документация/разрешения, зарплаты, коммунальные, транспорт, реклама, прочие
- **Переводы:** между счетами/кассами
- Каждая операция автоматически конвертируется в основную валюту по текущему курсу

#### 4.6.2. Мультивалютная система

> ⚠️ Критически важный функционал для рынка Таджикистана

- Настраиваемый список валют: TJS (сомони), USD (доллар), RUB (рубль), EUR (евро) и др.
- Основная валюта выбирается в настройках компании (все отчёты формируются в ней)
- Курсы валют вводятся **вручную** (ежедневно или по мере изменения)
- История курсов сохраняется
- При совершении любой операции в иностранной валюте, система автоматически фиксирует курс и пересчитывает в основную валюту
- Курсовые разницы учитываются при изменении курса
- Возможность автоматического подтягивания курсов от ЦБ (в будущем)

#### 4.6.3. Расчётные счета и кассы

Компания может иметь несколько расчётных счетов и касс:

- **Банковские счета** — название банка, номер счёта, валюта, текущий баланс
- **Наличные кассы** — название, ответственный, валюта, текущий баланс
- **Мобильные кошельки** — если применимо
- Для каждого счёта/кассы: текущий баланс с разбивкой по валютам
- Переводы между счетами с указанием суммы, валюты, курса конвертации
- История всех операций по каждому счёту
- Автоматическая сверка: баланс = начальный + все доходы – все расходы

#### 4.6.4. Категории расходов

Предустановленные и настраиваемые категории расходов для строительной компании:

- Приобретение земельного участка
- Проектирование и документация
- Разрешения и лицензии
- Стройматериалы (связь со складским модулем)
- Оплата мастеров и подрядчиков (связь с модулем мастеров)
- Заработная плата сотрудников
- Аренда техники и оборудования
- Транспортные расходы
- Коммунальные расходы на стройплощадке
- Маркетинг и реклама
- Административные расходы
- Налоги и сборы
- Прочие расходы

#### 4.6.5. Финансовые отчёты

- **Отчёт о прибылях и убытках (P&L)** — по объекту и по компании
- **Баланс** — активы (квартиры, деньги) vs обязательства (долги, рассрочки)
- **Cash Flow** — движение денежных средств по периодам
- **Дебиторская задолженность** — кто сколько должен, когда ближайший платёж
- **Кредиторская задолженность** — кому компания должна (мастера, поставщики)
- **Себестоимость объекта** — все расходы на землю + стройку + документацию
- **Маржинальность** — доход от продаж – себестоимость

### 4.7. Модуль «Мастера и подрядчики»

#### 4.7.1. Реестр мастеров

Отдельная вкладка для учёта всех строительных мастеров и бригад:

- Тип: одиночный мастер или бригада
- Для бригады: бригадир (контактное лицо) + список членов бригады
- ФИО, телефон, паспортные данные
- Специализация: штукатурка, электрика, сантехника, плиточные работы, кровля, фасад и т.д.
- Рейтинг качества (внутренний, от прораба)
- История работ: на каких объектах/этажах работал

#### 4.7.2. Наряды и работы

Для каждого мастера/бригады фиксируются наряды:

- Объект и локация (блок, этаж, квартиры)
- Вид работ (из справочника)
- Единица измерения: м², погонный метр, штука, комплект
- Объём работ
- Цена за единицу
- Общая стоимость наряда (автоматический расчёт)
- Дата начала и окончания работ
- Статус: назначен, в работе, выполнен, принят прорабом, оплачен
- Акт выполненных работ (генерируется из системы)

#### 4.7.3. Оплата мастеров

- Общая начисленная сумма по всем нарядам
- Аванс (предоплата)
- Промежуточные выплаты
- Окончательный расчёт
- Удержания (за брак, материалы и т.д.)
- **Баланс: начислено – выплачено = задолженность/переплата**
- Все выплаты автоматически попадают в журнал финансов как категория «Оплата мастеров»

### 4.8. Модуль «Склад и стройматериалы»

#### 4.8.1. Справочник материалов

- Название материала
- Категория: бетон, арматура, кирпич, цемент, отделочные, электрика, сантехника, прочее
- Единица измерения: тонна, м³, м², штука, упаковка
- Артикул (внутренний)

#### 4.8.2. Приход и расход материалов

- **Приход:** поставщик, накладная, дата, количество, цена за единицу, общая сумма, объект
- **Расход:** объект, блок/этаж (куда списано), количество, дата, ответственный
- **Перемещение:** между объектами/складами
- Текущий остаток по каждому материалу на каждом объекте
- Минимальный остаток — оповещение при достижении (уведомление завскладу и бухгалтеру)

#### 4.8.3. Работа с поставщиками

- Реестр поставщиков: название, контакты, специализация, рейтинг
- История закупок у каждого поставщика
- Кредиторская задолженность перед поставщиками

Настраиваемый дашборд с ключевыми метриками компании:

#### 4.10.3. Фильтры и экспорт

## 7. Дополнительные возможности

### 7.1. Динамическое ценообразование

Система автоматического изменения цен на квартиры на основе правил:

- Повышение цены при достижении порога продаж (например, каждые 10% проданных квартир = +5% к цене)
- Повышение цены по мере готовности этажа
- Скидки на последние квартиры определённого типа
- Сезонные акции
- Все изменения цен логируются

### 7.2. Агентский канал

Работа с внешними брокерами и риелторами:

- Регистрация агентов и агентств-партнёров
- Личный кабинет агента: просмотр шахматки, бронирование
- Агентское вознаграждение: процент от сделки, настраивается по агенту/объекту
- Автоматический расчёт и учёт вознаграждений
- Статистика по агентам: количество сделок, сумма, эффективность

### 7.3. Учёт хода строительства

- Этапы строительства по объекту: фундамент, каркас, фасад, инженерные сети, отделка, сдача
- Процент готовности каждого этажа/блока
- Фотоотчёты с привязкой к дате
- Камеры наблюдения (стриминг — в будущем)
- Уведомления клиентам о прогрессе строительства

### 7.4. Калькуляторы

- **Калькулятор рассрочки:** ввод цены, первого взноса, срока → автоматический график
- **Калькулятор ипотеки:** ставка, срок, первый взнос → ежемесячный платёж
- **Калькулятор себестоимости квартиры:** расход на землю + стройку / площадь

### 7.5. Логирование и аудит

- Аудит-лог всех действий: кто, когда, что изменил
- Логирование входов в систему
- Отслеживание изменений цен, статусов квартир, платежей
- Невозможность удаления данных — только мягкое удаление (soft delete)

### 7.6. Уведомления внутри системы

- Уведомления в реальном времени (WebSocket)
- Типы: новая заявка, новый платёж, просроченный платёж, задача, системное
- Push-уведомления на мобильное устройство
- Email-уведомления (для критичных событий)

## 10. Глоссарий терминов

| Термин | Определение |
|--------|-------------|
| **SaaS** | Software as a Service — программное обеспечение как услуга, доступ через интернет по подписке |
| **Тенант** | Отдельная строительная компания (арендатор) в мультитенантной системе |
| **ЖК** | Жилой комплекс — объект строительства |
| **Шахматка** | Визуальная сетка/матрица всех квартир объекта с цветовой индикацией статусов |
| **Воронка продаж** | Последовательность этапов, через которые проходит клиент от первого контакта до покупки |
| **CRM** | Customer Relationship Management — система управления взаимоотношениями с клиентами |
| **RBAC** | Role-Based Access Control — управление доступом на основе ролей |
| **RLS** | Row Level Security — безопасность на уровне строк в базе данных |
| **Бронь** | Временная резервация квартиры за клиентом (с ограниченным сроком) |
| **Рассрочка** | Оплата стоимости квартиры частями по графику платежей |
| **Наряд** | Задание для мастера/бригады на выполнение определённых работ |
| **Soft Delete** | Мягкое удаление — данные помечаются как удалённые, но не стираются из базы |
| **MVP** | Minimum Viable Product — минимально жизнеспособный продукт |
| **API** | Application Programming Interface — интерфейс программирования приложений |
| **Динамическое ценообразование** | Автоматическое изменение цен на основе заданных правил и условий |

## 11. Заключение

Данное техническое задание описывает комплексную SaaS-платформу для автоматизации бизнес-процессов строительных компаний. Платформа охватывает все ключевые аспекты работы застройщика: от приобретения земли до продажи квартир и послепродажного обслуживания клиентов.

Система разрабатывается с учётом лучших практик конкурентов (Profitbase, Домопланер, JoyWork, SberCRM и др.), а также специфики рынка Таджикистана: мультивалютность, местные SMS-провайдеры, учёт бартерных сделок при покупке земли.

Рекомендуемый подход к разработке — поэтапный (Agile), с выпуском MVP за 3–4 месяца и последующим расширением функционала. Это позволит оперативно начать работу с первыми клиентами (застройщиками) и итеративно улучшать продукт на основе обратной связи.

Документ является основой для планирования разработки, оценки трудозатрат, постановки задач команде и контроля качества. При необходимости каждый модуль может быть детализирован в отдельном документе.
